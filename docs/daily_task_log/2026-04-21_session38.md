## 교훈 / 메모

- `receive_sharing_intent` 1.8.1 은 Kotlin jvmTarget=17 고정 → 루트 `android/build.gradle.kts` 에서 subprojects JVM 정렬이 필요. 같은 패턴 plugin 추가 시 Gradle 오류 발생 가능성 인지.
- Cold-start one-shot seed 는 Riverpod Notifier 의 `setInitial` + `consume` 로 해결. Redirect 함수는 동기라 `ref.read`/`notifier.consume` 만 사용. 비동기 stream 은 Phase 1 후반에 별도 provider 로 도입 예정.
- GoRouter redirect 에서 pending URL consume 시 다음 루프에는 null → 무한 redirect 루프 없음 (state 변화만으로는 redirect 재실행 트리거 아님. 라우트 변경이 실제 원인).
- LinkAddScreen prefill 은 `initState` 에서 controller 동기 세팅 + post-frame callback 으로 `updateUrl` — `linkFormProvider` 가 AsyncNotifier 라 build 완료 전 notifier 호출 시점 보장 필요.
