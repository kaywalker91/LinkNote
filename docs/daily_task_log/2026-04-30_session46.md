## 변경 요약

### Code

**`lib/shared/widgets/link_list_tile.dart`**
- 신규 파라미터 `String? highlightText`. `null` / 빈 문자열 / 매칭 0건 → 기존 `Text` 동일.
- 매칭 시 `Text.rich` + `TextSpan` 으로 분할. 매칭 토큰만 `backgroundColor: AppColors.forestSoft` + `color: AppColors.forestInk` + `fontWeight: w600`.
- 제목과 `_displayUrl(host)` 양쪽에 적용. `_buildHighlightedSpans` 헬퍼는 case-insensitive lower-cased indexOf 반복.

**`lib/features/search/presentation/screens/search_screen.dart`**
- `Scaffold(backgroundColor: AppColors.bg)`.
- `AppBar` → `backgroundColor: AppColors.bg` + `surfaceTintColor: AppColors.bg` + `elevation: 0` + `shape: bottom BorderSide(line)`.
- 검색 입력은 `Container(decoration: bgSunk + radius lg + line border)` 에 `Icons.search`(ink3) + `TextField`(cursorColor forest, ink/ink3 톤, isDense) + `LnIconBtn(close_rounded, ink3)` 조합.
- 입력 폼 자체를 `AppBar.title` 안의 `Padding` 에 넣어 `screenPadding` 좌우 정렬.
- `_SearchBody`: 결과 `LinkListTile` 에 `highlightText: state.query` 전달.
- 최근 검색 라벨/버튼 톤: `AppTextStyles.label` + `AppColors.ink2/forest` 로 정렬.

### Tests

**신규 `test/shared/widgets/link_list_tile_highlight_test.dart`** (4 cases, RED→GREEN)
- `highlightText null` → 기존 `Text` 경로.
- `highlightText 'forest'` (case-insensitive) → `RichText` 의 `TextSpan` 자식 중 `text == 'Forest'` + `backgroundColor == AppColors.forestSoft` 발견.
- 빈 문자열 → 하이라이트 안 함.
- 미매치 → 원본 텍스트 보존.

**`test/integration/search_to_detail_flow_test.dart`**
- clear 아이콘 finder: `Icons.clear` → `Icons.close_rounded` (위젯 변경 정합성).
