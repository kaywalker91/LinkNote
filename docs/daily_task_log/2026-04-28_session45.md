## 학습 / 교훈

- **`gh pr checks` 출력 파싱은 JSON 으로**: 탭 구분 + 공백 포함 컬럼 (`"Build (Android Dev Debug)"`) 때문에 awk whitespace split 가 침묵 실패하기 쉬움. 모니터링/스크립트는 `--json name,bucket` 같은 안정적 키로 정렬할 것. (모니터 silence 가 success 와 구분 안 되는 함정과 동일 카테고리 — guard 가 적극적으로 매칭해야 함.)
- **`on Exception catch` → `on Object catch` 일관성**: Session 28 (link_local DS), Session 41 (link/remote DS catch 12곳, PR #22), Session 44 PR #26 (search remote DS) 에 이어 본 세션에서도 Collection 삭제 플로우 1곳 정렬. 데이터/UI 경계의 catch 절은 일관되게 `Object` 로 — `feedback_dart_error_vs_exception.md` 정책 답습.
- **단일 사용자 i18n 카피 변경은 셀렉터 sweep 필수**: 화면 카피 한글화 직후 위젯 + 통합 테스트 셀렉터 동기화를 sweep 하지 않으면 회귀가 첫 `flutter test` 에서 노출. 본 세션은 Stage 1/2 코드 변경 직후 같은 PR 에 셀렉터 동기화 포함 → 분리 PR 회피.
