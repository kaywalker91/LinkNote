## 변경 요약

### Code

**신규 `lib/shared/widgets/ln/ln_collection_card.dart`**
- 그라디언트 헤더 카드. 78px 헤더 + folder 아이콘 + 이름 + "링크 N개".
- `enum LnCollectionTone { forest, lilac, slate, amber }` 4톤. `gradient` extension getter 가 핸드오프 `palette` (`#4E9C7B→#1F6E53`, `#A595C9→#4F4370`, `#526578→#2E3A47`, `#E0A25A→#8F5B1F`) 그대로 노출.
- `static LnCollectionTone toneForId(String id)` — `id.codeUnits` 31진법 해시 % 4. 같은 id 는 항상 동일 톤(deterministic palette).
- `Material + InkWell` 컨테이너, `BorderRadius AppRadius.lg`, border `AppColors.line`. 헤더는 `LinearGradient(topLeft → bottomRight)` + `Align(bottomLeft) + Icons.folder_rounded(white, size 24)`.
- 본문 패딩 `EdgeInsets.fromLTRB(md, sm+2, md, md)`. 이름 `AppTextStyles.titleM(ink, w600, 2 lines, ellipsis)`. 카운트 `AppTextStyles.caption(ink3)`.

**`lib/features/collection/presentation/screens/collection_list_screen.dart`**
- `PaginatedListView`(ListView 기반) → `CustomScrollView + SliverGrid` 인라인. 2열 grid (`crossAxisCount: 2`, `crossAxisSpacing: AppSpacing.md`, `mainAxisSpacing: AppSpacing.md`, `childAspectRatio: 0.95`).
- 패딩 `EdgeInsets.fromLTRB(AppSpacing.md+2, AppSpacing.md, AppSpacing.md+2, 0)` + 하단 `huge` 여백 SliverToBoxAdapter (FAB overlap 방지).
- `_PaginatedGrid` StatefulWidget — `ScrollController` 리스너로 `position.pixels >= maxScrollExtent - 200` 시 `onLoadMore()` 호출. `RefreshIndicator` 래핑 + `state.hasMore && state.isLoadingMore` 푸터 `CircularProgressIndicator`.
- `_SkeletonGrid` StatelessWidget — 동일 grid delegate, `itemCount: 6` 의 `CollectionCardSkeleton` 셀.
- 빈 상태 (`state.items.isEmpty`) 는 grid 외부에서 `EmptyStateWidget` 직접 분기 — Sliver 안에 두지 않아 표시 영역 가운데 정렬 유지.
- LnTopBar / 컬렉션 추가 LnIconBtn / `collections_fab` (endFloat) FAB — 변경 없음. shell 의 `shell_fab` (centerDocked) 와 위치/heroTag 충돌 없음.

**`lib/shared/widgets/skeleton/collection_card_skeleton.dart`**
- 그리드 셀 카드 형태로 재구성. 외부 패딩 제거(셀이 알아서 가짐). `DecoratedBox(border AppColors.line, radius lg) + Column(헤더 78 ShimmerBox + 텍스트 라인 120/64)`.
