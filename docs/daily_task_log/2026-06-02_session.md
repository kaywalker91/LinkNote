## 오늘의 학습

### 새로 배운 것
- **검증 순서 = 진단 품질**: 여러 validation이 동시 위반될 때 "가장 근본적인 결함(unidentified entity)"을 먼저 반환하면, 단독 위반 동작은 불변이면서 동시-위반 케이스의 진단성이 올라간다. 단독 가치는 cosmetic이지만 진단 표면 개선은 실질.
- **clock seam은 의존성 없이**: `DateTime Function()? now` optional 파라미터(default `DateTime.now`)는 `clock` 패키지 같은 무거운 추상화 없이 결정론적 경계 테스트를 가능케 하는 최소 seam. `const` 생성자는 포기해야 하지만 DI가 `const` 없이 생성하면 무영향.
- **자동 분석가 보고는 검증 후 채택**: 감사 에이전트가 "DateRangePicker 실제 크래시"를 high-confidence로 보고했으나, 실제론 `GlobalMaterialLocalizations.delegate`가 이미 등록되어 있어 오탐. **falsifiable claim은 grep 한 번으로 확인** → 없는 버그 고치는 헛수고 방지(Epistemic Honesty 룰의 성공 사례).
