## Session 33 lessons

### Riverpod `AsyncNotifier` in-flight state wrapping

- 비동기 메서드(Future<void>) 호출 중에는 외부에서 읽은 state가 `AsyncLoading<T>()`(no previous)로 래핑되어, optimistic 업데이트를 mid-flight에 관찰할 수 없음
- **How to apply**: optimistic 패턴 테스트는 "완료 후 최종 상태"만 검증. mid-flight 관찰을 위해 Completer를 사용해도 외부 read는 AsyncLoading만 보임
- deleteLink/toggleFavorite 테스트도 동일 한계가 있으며 같은 패턴(완료 후 검증)으로 작성됨

### 머지 후 로컬 main upstream tracking

- squash PR 머지 후 로컬 main이 `upstream to origin/main` 설정이 끊긴 상태(filter-repo 흔적)가 있음
- **How to apply**: 매 세션 시작 시 `git branch -vv`로 upstream 확인. 필요 시 `git branch --set-upstream-to=origin/main main`
