## 학습/교훈

- **`keepAlive: false` 명시의 한계**: `avoid_redundant_argument_values` lint 와 충돌. "명시"의 의도를 살리려면 기본 annotation(`@riverpod`) + docstring 주석으로 우회. 프로젝트에서 `// ignore:` 를 금지하므로 이 방식이 유일한 답
- **ref.listen 으로 controller 미러링 패턴**: stateful widget 에 TextEditingController 가 있고 provider 상태를 반영해야 할 때, `_syncControllers(state)` 를 빌드 함수에서 호출하는 기존 패턴(초기 1회)에서 `ref.listen((prev, next) { if (text != value) controller.value = ... })` 패턴(지속 미러링)으로 전환하면 프로그래매틱 업데이트도 자동 반영됨. cursor 보존 로직 포함 필요
- **"명시적 이월" 의 가치**: P3-A 를 해결 없이 리뷰 문서에 "Deferred (Session 35)" 표시 + PRD 선결 과제 기록. 다음 세션에서 리뷰 문서 열면 "무엇이 막혔나" 가 바로 보이므로 재시작 오버헤드가 낮아짐
