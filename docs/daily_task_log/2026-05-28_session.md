## 오늘의 학습

### 새로 배운 것
- **정보 노출 표면 분석**: `Failure.message`에 raw 예외를 담아도, `failure_ui.dart`가 대부분 고정 문자열로 매핑하면 release에선 거의 안 새고 실제 누출은 `AuthFailure(:message)` passthrough와 `kDebugMode` 경로뿐 — "위험 표면"을 코드로 좁혀 확인하면 over-engineering 없이 정밀한 범위 설정이 가능.
- **TDD 친화적 sweep 스코프**: typed 예외 분기(`PostgrestException`/`AuthException` `e.message`)와 generic `Object` fallback을 구분하면, 메시지를 단언하는 테스트(대부분 typed 분기 검증)를 깨지 않고 fallback만 안전하게 일괄 변경 가능 — "테스트 11개 깨짐" 우려가 실제론 ≈0이었던 이유.
- **grep 가드 FP 경계 설계**: `\$e[^[:alnum:]_]` 로 `$events` 같은 합법 보간을 제외하고 `e.message`(typed)도 제외해야 깨끗한 anti-pattern 가드가 됨. 구 Track I의 toJson 검출이 infeasible했던 것과 달리, raw `$e`/`e.toString()`는 grep-feasible.
- **외부 LLM 교차 리뷰의 가치**: Codex와 agy가 F5 잔존 여부에서 의견이 갈림(agy "중립화됨" 낙관 vs Codex+실측 "여전히 열림") — 단일 모델 맹신 대신 실코드 grep으로 판정.
