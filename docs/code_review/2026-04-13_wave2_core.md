### P0-D: Result.when() — 비-nullable T에서 null 데이터 시 런타임 TypeError

**File**: `lib/core/error/result.dart:14`

```dart
return success(data as T);
```

**문제**: `Result<T>`의 `when()` 메서드에서 `data as T` 캐스트. `isSuccess`가 `failure == null`로 판단되므로, 만약 `failure == null && data == null`인 레코드가 생성되면 비-nullable `T`에 대해 `null as T`가 sound null-safety 런타임 TypeError를 발생시킴. 현재 코드베이스에서 `when()`은 거의 사용되지 않고 `.isSuccess`/`.data!` 패턴을 주로 사용하므로 즉각적 크래시 가능성은 낮으나, 공개 API로 존재하는 이상 잠재적 위험.

**권장 픽스**: `when` 콜백 시그니처를 `T?`로 변경하거나, `data!`로 명시적 null 체크.

### P1-E: 비동기 콜백 타입 불일치 — VoidCallback에 async 할당

**File**: `lib/core/network/auth_interceptor.dart:9` + `lib/core/network/dio_client.dart:29`

**문제**: `AuthInterceptor.onUnauthorized`는 `VoidCallback?` (= `void Function()?`) 타입. `dio_client.dart`에서 `() async { ... }` 클로저를 할당하면 반환되는 `Future`가 버려짐. `signOutUsecaseProvider.call()`이 unawaited 실행되어 캐시 정리 완료 전 UI 전환 가능.

**참고**: P1-C에서 이 provider가 미사용이므로 현재 런타임 영향 없음. 그러나 이 provider를 활성화하면 즉시 문제 발생.

**권장 픽스**: `VoidCallback?` → `Future<void> Function()?`로 타입 변경.

## Positive Patterns (잘된 점)

- **Result<void> 가드**: `auth_provider.dart:96`에서 `!result.isFailure` 사용 (isSuccess 대신). 의도적이고 안전.
- **keepAlive 규율**: 5개 provider만 keepAlive 사용 (connectivity, isOnline, sessionExpired, themeMode, appRouter). Feature provider에 잘못 적용된 케이스 없음.
- **Failure sealed class 망라성**: `failure_ui.dart`의 switch가 모든 5개 variant 커버. default 폴백 없음.
- **withValues(alpha:) 일관성**: `.withOpacity()` (deprecated) 미사용. 모든 알파 블렌딩에 최신 API 사용.
- **LoggingInterceptor kDebugMode 게이트**: 프로덕션 빌드에서 로깅 비활성화 확인.
- **OgTagService 분리 Dio**: 인증 토큰 없는 별도 Dio로 외부 서버에 토큰 유출 방지.
- **Hive 암호화 일관성**: 모든 box에 `encryptionCipher` 적용. 평문 box 없음.
- **AuthInterceptor 관심사 분리**: 인터셉터 내부에 비즈니스 로직 없이 콜백 위임.
